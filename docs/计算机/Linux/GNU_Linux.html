什么是GNU/Linux ​

  • GMU是一套操作系统，是一组供用户使用的软件程序：
    • GNU名字的含义是一个递归缩写，代表“GNU's Not Unix”（GNU不是Unix）。这个名称由自由软件基金会（Free Software Foundation, FSF）的创始人理查德·斯托曼（Richard Stallman）提出，旨在创建一个完全自由的Unix兼容操作系统，目标是开发一个自由的操作系统，强调用户在使用软件时应享有自由，包括运行、复制、分发、学习、修改和改进软件的自由。
    • GNU操作系统由许多自由软件组成，这些软件遵循GNU通用公共许可证（GPL），这是一种广泛使用的自由软件许可证。因此GNU不仅是一个操作系统，它还包含大量的自由软件包，如编译器GCC、文本编辑器Emacs等，这些软件都是由GNU项目或第三方发布的。通过这些软件，用户可以自由地使用计算机而不必担心侵犯版权或受到商业软件的限制。
  • Linux也是操作系统，但是最底层的软件，直接与硬件交互，称为操作系统的内核部分，负责管理硬件资源和为应用程序提供底层服务。
  • Linux与GNU项目有着密切的关系：尽管Linux内核本身并不构成一个完整的操作系统，但其与GNU项目的结合使得它能够提供完整的系统功能，Linux内核负责硬件管理，而GNU工具链和软件则提供了用户界面、编程工具和其他应用程序。
  • GNU操作系统通常与Linux内核结合使用，诞生了世界上最自由的 GNU/Linux操作系统。
  • Linux发行版（如Ubuntu、Debian、Fedora等）通常被称为“Linux”，但实际上它们更准确地应该被称为“GNU/Linux”，因为它们依赖于GNU项目提供的大量软件。

Linux的结构 ​

Linux可划分为以下四部分：

  1. Linux内核：内核控制着计算机系统上的所有硬件和软件

    • 系统内存管理
    • 软件程序管理
    • 硬件设备管理
    • 文件系统管理

  2. GNU工具：一些系统工具，用于执行一些标准功能，比如控制文件和程序。通常将Linux内核和GNU工具的结合体称为Linux

    • GNU coreutils软件包（核心GNU工具）由三部分构成：
      • 用以处理文件的工具
      • 用以操作文本的工具
      • 用以管理进程的工具
    • shell：壳程序shell是用户操作系统的一个界面或平台，是一种特殊的交互式工具，就是命令行模式。它为用户提供了启动程序、管理文件系统中的文件以及运行在Linux系统上的进程的途径。shell的核心是命令行提示符。命令行提示符是shell负责交互的部分。它允许你输入文本命令，然后解释命令，并在内核中执行。
      • shell包含了一组内部命令，用这些命令可以完成诸如复制文件、移动文件、重命名文件、显示和终止系统中正运行的程序等操作。
      • shell也允许你在命令行提示符中输入程序的名称，它会将程序名传递给内核以启动它。
      • 也可以将多个shell命令放入文件中作为程序执行。这些文件被称作shell脚本。你在命令行上执行的任何命令都可放进一个shell脚本中作为一组命令执行。这为创建那种需要把几个命令放在一起来工作的工具提供了便利。
      • Linux系统上，通常有好几种Linux shell可用。在/etc/shell目录下所有Linux发行版默认的shell都是bash shell。
        • bash shell由GNU项目开发，被当作标准Unix shell——Bourne shell（以创建者的名字命名）的替代品。
        • 其他shell的简单介绍 | 其它shell | 描述 | | --------- | -------------------------------------------------------- | | ash | 一种运行在内存受限环境中简单的轻量级shell, 但与bash shell完全兼容 | | korn | 一种与Bourne shel1兼容的编程sell,但支持如关联数组和浮点运算等一些高级的编程特性 | | tcsh | 一种将c语言中的一些元素引入到shell脚本中的shell | | zsh | 种结合了bash. tcsh和korn的特性, 同时提供高级编程特性、共享历史文件和主题化提示符的高级shell |

  3. Linux桌面环境（图形化界面GUI）：相对于文本界面CLI

    • 这一部分是可选的，并不是所有的程序都需要图形化界面，Linux中相当一大部分程序可以通过命令行界面与用户交互，例如[GROMACS]
    • KDE桌面：（K Desktop Environment，K桌面环境）类似于Microsoft Windows的图形化桌面环境，例如 openSuSE Linux发行版
    • GNOME桌面：（the GNU Network Object Model Environment，GNU网络对象模型环境），例如 CentOS 7 和Ubuntu20 Linux发行版

  4. 应用软件：通过GNU工具和Linux桌面环境运行在Linux内核之上，是Linux系统直接与用户交互的一部分

Linux的软件管理 ​

软件包和包管理器 ​

  • Linux中很多软件都是以源代码方式进行发布的，但是源代码对普通用户来讲编译成一个可执行的文件过程是非常繁琐的。所以有些软件商帮我们把互联上的开源软件编译成了二进制并且把它打包成一个文件然后我们可以直接用包管理器将软件商打包好的文件直接进行安装。
  • 安装软件的两种方式
    • 从源码（编程代码）自行编译
    • 通过软件包管理系统，已经编译好的二进制安装程序
      • 软件包管理系统提供安装、更新、删除及卸载软件的方法，同时提供对系统中所有软件状态信息的查询。不同的Linux发行版会有不同的包管理器，比较常见的包管理工具有DPKG、APT(APT-GET)、RPM、YUM、DNF、PACMAN等

常见的软件包格式 ​

  1. Debian系列：.deb格式文件, dpkg、apt包管理器
  2. Red hat系列：.rpm格式文件, rpm、yum、dnf 包管理器，rpm(Redhat Package Manager)后来改名成RPM Package Manager

软件包命名方式：name-version-release.arch.rpm
例如：akonadi-1.9.2-4.el7.x86_64.rpm

  • akonadi：为name包名
  • 1.9.2：为version版本信息：主版本号.次版本号.修正号
  • 4：二进制包发布的次数，表示此RPM包是第几次编译生成的
  • el7：软件发行商，el7表示此包是由Red Hat公司发布，适合在RHEL7.x和Centos7.x上使用
  • x86_64：为运行平台，此参数可以看到该软件包可用于什么平台上
  1. 其他安装包格式：
    • tar.gz包：这种格式的包通常包含源代码，可以通过命令tar -zxvf 包名.tar.gz解压。解压后，通常会有一个名为Install的纯文本文件，指导用户进行安装。此外，解压后还可能有.configure、.make和.make install等文件，用于编译和安装软件。
    • bin包：这种格式的包通常包含二进制可执行程序，如.bin或.run后缀的文件。
    • run包：这种格式的包与bin包类似，安装时也需要给包加上可执行权限
  2. 软件包的获取、软件源仓库：软件包需要事先将源代码进行编译后打包形成，除了在发行版官方的仓库中找，获取包的途径如下：华为、阿里云、网易、大学等开源镜像

安装位置 ​

  • 主要的安装位置有两个，分别是 /opt 目录和 /usr/local 目录。
  • 在ubuntu系统中，下载的安装包被存放在/var/cache/apt/archives目录下。当我们使用apt-get或apt命令来安装软件时，系统会自动从这个目录中查找相应的包，并进行安装。这个目录是系统默认的软件包缓存目录，它保存了我们已经下载过的包，以便将来再次安装时能够快速找到。/var/cache/apt/archives目录下的安装包都是以.deb为后缀的文件，这是ubuntu系统的软件包格式。
  • 通过在终端中运行命令sudo apt-get install 软件包名称，可以从 Ubuntu 软件仓库中直接下载和安装软件。这些软件包会被安装在系统的“/usr”目录下的特定位置，具体位置会依据软件的类型和用途而有所不同。
  • 一些软件可以通过第三方软件源进行安装，比如 PPA（Personal Package Archives）或者通过软件开发者的官方网站提供的下载链接，并使用适当的命令进行安装，软件包会被存储在系统的“/opt”目录下，通常会创建一个与软件名称相对应的文件夹。
  • 用户也可以通过直接下载软件包并手动安装的方式来获得软件。这些软件包可能是通用的二进制文件，也可能是经过编译的源代码。用户可以选择将软件包存储在任意位置，然后使用相应的命令进行安装。
  • 通过包管理器安装的软件通常会在系统的“/usr/bin”目录下创建一个可执行文件。用户可以在终端中直接调用该文件来启动软件。此外，一些软件还会在“/usr/share/applications”目录下创建一个 .desktop 文件，这样用户就可以在图形界面中更方便地启动软件了。(ubuntu系统在~/.local/share/applications/)
  • 如果是自行从源码编译安装的程序，通常不会将可执行文件放在“/usr/bin”目录，要在终端中直接调用需要添加环境变量

环境变量 ​

  • 环境变量用于指定操作系统运行环境的一些参数，如PATH、HOME、USER、HISTSIZE、LOGNAME、HOSTNAME、SHELL、LANGUAGE、MAIL和PS1等。这些变量可以在系统级别或用户级别设置，并且可以分为永久性和临时性两种。系统环境变量对系统中所有用户都有效，而用户环境变量只对特定用户有效。
  • 用户可以通过编辑个人配置文件（如~/.bashrc或~/.profile）来设置用户的环境变量，这些文件通常位于用户的主目录下。
  • 系统环境变量通常设置在/etc/profile文件中，这个文件在用户登录时会自动读取，以设置系统级的环境变量。
  • 拓展阅读：linux环境变量

运行软件方式 ​

  • 除了系统自带的可执行程序，直接输入自己的程序、新安装的程序，系统没办法获取到该执行程序的所在文件路径，即系统不知道这个程序在哪里，无法运行。
  • 打印一下当前系统 PATH 环境变量：echo $PATH
  • 将要执行的文件所在的路径添加到该变量中：PATH=$PATH:可执行程序的文件所在路径
